## ⚠️ 重要：配置 WebSocket

当前 MQTT Broker 只支持 TCP (1883)，Web 浏览器需要 WebSocket (8083)。

### 快速解决方案：安装 EMQX

1. **下载 EMQX**
   ```
   https://www.emqx.io/downloads
   选择: Windows AMD64
   ```

2. **解压并启动**
   ```bash
   # 解压到任意目录，例如 C:\emqx
   cd C:\emqx\bin
   emqx.cmd console
   ```

3. **验证端口**
   ```bash
   netstat -an | findstr "8083"
   # 应该看到: TCP    0.0.0.0:8083
   ```

4. **刷新浏览器**
   - 左下角应显示: **🟢 已连接**

### 1. 查看实时数据
- HUD 面板显示设备状态
- 初始状态: 停止 (RPM=0)

### 2. 启动机器
- 点击右侧 **"启动机器"** 按钮
- 观察:
  - 运行状态变为 "运行中" (绿色)
  - RPM 开始变化 (1000-1500)
  - 螺杆开始旋转
  - 温度上升

### 3. 停止机器
- 点击 **"停止机器"** 按钮
- 观察:
  - 运行状态变为 "停止" (红色)
  - RPM 归零
  - 螺杆停止旋转

### 4. 设置转速
- 输入框输入转速 (例如: 1800)
- 点击 **"设置"** 按钮
- 观察 RPM 变化

## 下一步：导入真实 3D 模型

当前使用的是简化的测试模型。要使用真实模型：

### 方案 1: 从 UE4 导出 GLTF
```
1. 在 UE4 中选择 BP_Machine
2. File -> Export -> GLTF
3. 保存为 machine.gltf
4. 放到 aircity-web/public/models/
5. 修改 index.js 中的 loadMachineModel() 函数
```

### 方案 2: 使用原始 FBX/OBJ
```
1. 找到原始的 3D 模型文件 (.fbx, .obj)
2. 使用 Blender 转换为 GLTF
3. 确保组件名称与代码中的一致
4. 导入到 Web 应用
```
